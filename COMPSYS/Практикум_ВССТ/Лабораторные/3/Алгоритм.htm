Алгоритм

 

МПС с индивидуальной памятью

 

В данном случае применяется один из методов «Управление ресурсами однопроцессорных систем оперативной обработки», т.е. SPT, RR или FB, но только для каждого из N процессоров данной МПС с индивидуальной памятью: создаются очереди для каждого процессора, все расчеты ведутся так же по каждому процессору в отдельности, а затем выводятся средние.

 

МПС с общей памятью

 

            При реализации работы МПС с общей памятью применяется один из методов «управления ресурсами многопроцессорных систем при обработке пакетов задач с прерываниями», т.е. Макнотона или LPT.

 

Примечание

 

         Для дальнейшего сравнительного анализа «МПС с индивидуальной памятью» и «МПС с общей памятью» рекомендуется использовать одинаковые входные параметры и одинаковые очереди заявок, т.е. сперва сгенерировать очереди заявок для «МПС с индивидуальной памятью» на N используемых процессоров, а затем объединить их в один массив заявок, для работы «МПС с общей памятью».
